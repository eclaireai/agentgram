# Turborepo Monorepo

> Web app + mobile + API + shared packages — one repo, 10x faster builds

## Why This Exists

Separate repos for web, API, and mobile means copy-pasted code, version mismatches, and no shared types. A monorepo with Turborepo gives you shared TypeScript types across all apps, cached builds (zero rebuild if unchanged), and one PR for cross-app changes.

## Steps (6 total)

1. ▶️ **Scaffold Turborepo with pnpm workspaces**
   `npx create-turbo@latest my-app --package-manager=pnpm`

2. 📄 **Create shared UI package with shadcn components exported as named exports**
   `packages/ui/package.json`

3. 📄 **Create shared database package with Drizzle schema and migrations**
   `packages/db/package.json`

4. 📄 **Create shared TypeScript config extended by all apps**
   `packages/config-typescript/tsconfig.base.json`

5. ✏️ **Configure pipeline: build depends on ^build, test is always run, dev has persistent: true**
   `turbo.json`

6. ▶️ **Enable Vercel remote cache — share build artifacts across team**
   `npx turbo login && npx turbo link`
