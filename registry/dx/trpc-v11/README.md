# tRPC v11 Type-Safe API

> End-to-end type-safe API — no codegen, no REST, just TypeScript

## Why This Exists

REST APIs require manual types on both sides that drift out of sync. GraphQL requires codegen. tRPC gives you end-to-end TypeScript types with zero overhead — change a server procedure's output and the client immediately shows a type error.
