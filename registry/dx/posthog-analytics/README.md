# PostHog Analytics + Feature Flags

> Event tracking, funnels, session replay, and feature flags — open source

## Why This Exists

Google Analytics tells you page views. PostHog tells you why users churn. Session replay shows you exactly where they got stuck. Feature flags let you ship to 5% of users before 100%. This is the analytics stack that makes products better.

## Steps (6 total)

1. ▶️ **Install PostHog browser SDK and Node.js SDK**
   `npm install posthog-js posthog-node`

2. 📄 **Create PostHog provider component with posthog-js initialization and user identification**
   `app/providers/PostHogProvider.tsx`

3. 📄 **Create track(), identify(), setUserProperties(), and page() wrapper functions**
   `lib/analytics.ts`

4. 📄 **Create isFeatureEnabled(flag, userId) using posthog-node for server-side flag evaluation**
   `lib/feature-flags.ts`

5. 📄 **Add Bootstrap data to middleware response for SSR feature flags without waterfall**
   `middleware.ts`

6. ✏️ **Add NEXT_PUBLIC_POSTHOG_KEY and NEXT_PUBLIC_POSTHOG_HOST**
   `.env.local`
