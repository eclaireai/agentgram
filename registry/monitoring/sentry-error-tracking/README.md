# Sentry Error Tracking

> Catch every unhandled error in production with full stack traces and context

## What it does

Integrate Sentry for error monitoring: unhandled exceptions, promise rejections, Express error middleware, user context, and custom breadcrumbs. Source maps uploaded on deploy so errors point to TypeScript lines.

## Why this recipe exists

You cannot fix errors you don't know about. Sentry captures every unhandled error in production with the full stack trace, the request that caused it, and who was affected — before your users file a support ticket.
