# RAG Pipeline with pgvector

> Production RAG: embed your docs, store in Postgres, semantic search in <100ms

## Why This Exists

RAG is the correct answer when your AI needs to know things that weren't in its training data (your docs, your database, your product). pgvector keeps vectors in your existing Postgres — no new infrastructure, no new vendor, no data leaving your stack.

## Steps (9 total)

1. ▶️ **Install AI SDK, ORM, and Postgres client**
   `npm install @ai-sdk/openai drizzle-orm pg`

2. ▶️ **Enable pgvector extension in your Postgres database**
   `psql -c "CREATE EXTENSION IF NOT EXISTS vector;"`

3. 📄 **Create documents table with content TEXT and embedding vector(1536) columns**
   `db/schema/documents.ts`

4. 📄 **Create embedText() function using text-embedding-3-small, returns Float32Array**
   `lib/embed.ts`

5. 📄 **Create ingest() that chunks documents, embeds each chunk, and upserts to DB**
   `lib/ingest.ts`

6. 📄 **Create semanticSearch(query, limit) using cosine similarity operator (<=>)**
   `lib/search.ts`

7. 📄 **Create RAG API route: search → inject context → stream answer with citations**
   `app/api/rag/route.ts`

8. 📄 **Create ingestion script that loads markdown files from /docs and ingests them**
   `scripts/seed-docs.ts`

9. ▶️ **Ingest sample documents and verify embeddings stored**
   `npx tsx scripts/seed-docs.ts`
   *Expected: N documents ingested*
