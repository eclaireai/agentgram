# Background Jobs with Inngest

> Durable async workflows that survive server restarts — no Redis, no Bull, no workers

## Why This Exists

Node.js setTimeout dies when your server restarts. Bull requires Redis. Inngest functions are durable by default — they survive crashes, retries are built-in, and you get a dashboard showing every job's history.

## Steps (8 total)

1. ▶️ **Install Inngest SDK**
   `npm install inngest`

2. 📄 **Create Inngest client with app ID and event key**
   `inngest/client.ts`

3. 📄 **Create sendWelcomeEmail inngest function triggered by user.created event**
   `inngest/functions/send-welcome-email.ts`

4. 📄 **Create processUpload function with multi-step: validate → transform → notify**
   `inngest/functions/process-upload.ts`

5. 📄 **Create Inngest route handler serving all functions**
   `app/api/inngest/route.ts`

6. ✏️ **Trigger user.created event after user registration**
   `app/api/users/route.ts`

7. ✏️ **Add INNGEST_EVENT_KEY and INNGEST_SIGNING_KEY**
   `.env.local`

8. ▶️ **Start Inngest dev server for local testing**
   `npx inngest-cli@latest dev`
   *Expected: Inngest dev server running*
