# Real-Time with Supabase

> Live cursors, presence, and database change streaming — no WebSocket server needed

## Why This Exists

Running a WebSocket server for a small app is massively over-engineered. Supabase Realtime is built on Phoenix Channels — battle-tested at massive scale — and you get it free with your Supabase project.

## Steps (6 total)

1. ▶️ **Install Supabase client (realtime is built in)**
   `npm install @supabase/supabase-js`

2. 📄 **Create useRealtimeTable(table) hook that subscribes to INSERT/UPDATE/DELETE and returns live rows**
   `hooks/useRealtimeTable.ts`

3. 📄 **Create usePresence(channelId) hook tracking online users with enter/leave events**
   `hooks/usePresence.ts`

4. 📄 **Create useBroadcast(channel) hook for sending/receiving arbitrary real-time messages**
   `hooks/useBroadcast.ts`

5. 📄 **Create live cursor overlay showing other users' mouse positions**
   `components/LiveCursors.tsx`

6. 📄 **Create presence indicator showing online count and avatars**
   `components/OnlineIndicator.tsx`
