# Pytest Setup

> Python testing with pytest, fixtures, coverage, and async support

## What it does

Set up pytest for a Python project: fixtures, conftest.py, async test support (pytest-asyncio), coverage reporting, and parametrize for data-driven tests.

## Why this recipe exists

pytest's fixture system is the cleanest dependency injection you'll find in any test framework. Fixtures compose, they're reusable, and they handle setup/teardown automatically. This recipe gets you to the good patterns immediately.

## Steps (6 total)

1. 🔍 **`find`** → `requirements.txt`  
   Check existing test dependencies

2. ⚡ **`run_command`** → `pip install pytest pytest-asyncio pytest-cov httpx`  
   Install pytest and essential plugins

3. 📄 **`create_file`** → `pytest.ini`  
   Configure pytest with asyncio_mode=auto, testpaths, and coverage settings

4. 📄 **`create_file`** → `tests/conftest.py`  
   Create shared fixtures: db session, test client, sample user

5. 📄 **`create_file`** → `tests/test_example.py`  
   Write first test using fixtures and parametrize

6. ⚡ **`run_command`** → `pytest --cov=src --cov-report=term-missing`  
   Run tests with coverage report
