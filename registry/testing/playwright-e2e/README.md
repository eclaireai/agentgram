# Playwright End-to-End Tests

> Cross-browser E2E testing with Playwright: Chrome, Firefox, Safari

## What it does

Add Playwright for end-to-end testing against a running server. Includes browser setup, page object model pattern, GitHub Actions CI integration, and screenshot on failure.

## Why this recipe exists

Unit tests tell you your functions work. E2E tests tell you your app works. Playwright is the modern choice — it's faster than Cypress, supports all browsers natively, and has first-class TypeScript support.

## Steps (9 total)

1. 🔍 **`find`** → `package.json`  
   Check existing test setup and dev server command

2. ⚡ **`run_command`** → `npm install -D @playwright/test`  
   Install Playwright test runner

3. ⚡ **`run_command`** → `npx playwright install chromium firefox webkit`  
   Install browser binaries

4. 📄 **`create_file`** → `playwright.config.ts`  
   Configure Playwright with baseURL, retries, screenshot on failure, and webServer

5. 📄 **`create_file`** → `e2e/pages/BasePage.ts`  
   Create BasePage class with common navigation and assertion helpers

6. 📄 **`create_file`** → `e2e/pages/HomePage.ts`  
   Create HomePage page object extending BasePage

7. 📄 **`create_file`** → `e2e/home.spec.ts`  
   Write first E2E test: visit homepage, check title, verify key element

8. ✏️ **`modify_file`** → `package.json`  
   Add e2e, e2e:ui, and e2e:debug scripts

9. ⚡ **`run_command`** → `npx playwright test`  
   Run E2E tests in headless mode
