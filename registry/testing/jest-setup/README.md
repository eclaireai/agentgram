# Jest Testing Framework

> Battle-tested unit testing with TypeScript, mocks, and Istanbul coverage

## What it does

Set up Jest with TypeScript (via ts-jest), Istanbul coverage, module mocking, and a sensible default config. The standard for Node.js testing.

## Why this recipe exists

Jest has the richest ecosystem: snapshot testing, mock functions, fake timers, and a debugger-friendly watch mode. When you need the full testing toolkit and don't need Vitest's speed, Jest is the standard.

## Steps (6 total)

1. 🔍 **`find`** → `package.json`  
   Check existing test setup and Node version

2. ⚡ **`run_command`** → `npm install -D jest ts-jest @types/jest`  
   Install Jest with TypeScript transformer

3. 📄 **`create_file`** → `jest.config.ts`  
   Create Jest config with ts-jest preset, coverage thresholds, and moduleNameMapper for path aliases

4. ✏️ **`modify_file`** → `package.json`  
   Add test, test:watch, and test:coverage scripts

5. 📄 **`create_file`** → `tests/example.test.ts`  
   Create sample test with jest.fn() mock demonstrating the pattern

6. ⚡ **`run_command`** → `npm test`  
   Verify tests run and pass
