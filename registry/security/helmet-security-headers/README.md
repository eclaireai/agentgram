# Security Headers with Helmet

> Add 15 security headers in one line — CSP, HSTS, XSS protection, and more

## What it does

Add Helmet.js to an Express app with a production-ready Content Security Policy, HSTS, and referrer policy. Configured correctly so it doesn't break your frontend.

## Why this recipe exists

By default, Express sends no security headers. Helmet adds HSTS, X-Frame-Options, X-Content-Type-Options, and a Content Security Policy. It takes 10 minutes and closes a dozen OWASP Top 10 vectors.

## Steps (6 total)

1. 🔍 **`find`** → `src/app.ts`  
   Check Express app setup and existing middleware order

2. ⚡ **`run_command`** → `npm install helmet`  
   Install Helmet

3. ✏️ **`modify_file`** → `src/app.ts`  
   Add helmet() as the first middleware (before routes), with custom CSP directives

4. 📄 **`create_file`** → `src/config/helmet.ts`  
   Create helmet config with CSP that allows your CDN and API origins, HSTS with 1-year maxAge

5. 📄 **`create_file`** → `tests/security.test.ts`  
   Test that responses include Strict-Transport-Security, X-Frame-Options, X-Content-Type-Options headers

6. ⚡ **`run_command`** → `npm test`  
   Verify security header tests pass
