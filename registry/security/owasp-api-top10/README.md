# OWASP API Security Top 10

> Harden your API against the 10 most exploited vulnerabilities in production

## Why This Exists

The OWASP API Top 10 are not theoretical — they are the exact vulnerabilities used in real breaches (Optus, Twitter, T-Mobile). This recipe adds 10 targeted defenses in order of exploitability. Each one takes 15 minutes; each skipped one is a liability.

## Steps (8 total)

1. ▶️ **Install security middleware**
   `npm install helmet express-rate-limit joi`

2. 📄 **Apply helmet() for all security headers (CSP, HSTS, X-Frame-Options, etc.)**
   `middleware/security.ts`

3. 📄 **Apply rate limiting: 100 req/15min globally, 5 req/15min on /auth routes**
   `middleware/rate-limit.ts`

4. 📄 **Create checkOwnership(req, resourceId) middleware for object-level authorization (BOLA fix)**
   `middleware/object-auth.ts`

5. 📄 **Validate all request bodies with Zod/Joi, reject unknown fields (mass assignment fix)**
   `middleware/input-validation.ts`

6. 📄 **Enforce max page size, max depth for nested queries, and request timeout**
   `middleware/query-limits.ts`

7. 📄 **Create parameterized query wrapper that prevents SQL injection**
   `lib/safe-query.ts`

8. 📄 **Create error handler that never leaks stack traces or internal details to API responses**
   `middleware/error-handler.ts`
