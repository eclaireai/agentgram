# CORS Configuration

> Precise CORS setup that works in dev, staging, and production without wildcards

## What it does

Configure CORS for an Express API: allowlist of origins per environment, preflight caching, credentials support, and logging for blocked origins to catch misconfigurations.

## Why this recipe exists

Using cors({ origin: '*' }) is a security hole. This recipe configures CORS with an explicit allowlist per environment — your frontend domain in production, localhost in dev — and logs rejected origins so you catch config issues fast.
