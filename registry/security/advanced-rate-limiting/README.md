# Advanced Rate Limiting (Upstash)

> Per-user rate limits with Redis — survives horizontal scaling and serverless

## Why This Exists

express-rate-limit uses in-memory storage — it breaks with multiple server instances and resets on deploy. Upstash Redis persists limits across instances, regions, and deploys. This is the correct implementation for production.

## Steps (5 total)

1. ▶️ **Install Upstash rate limiter and Redis client**
   `npm install @upstash/ratelimit @upstash/redis`

2. 📄 **Create rateLimiter with sliding window (10 req/10s) and separate limits for auth endpoints (5/60s)**
   `lib/rate-limit.ts`

3. 📄 **Create rate limiting middleware using limiter.limit(identifier) where identifier is IP or userId**
   `middleware/rate-limit.ts`

4. ✏️ **Apply rate limiting in Next.js middleware before route handling**
   `middleware.ts`

5. ✏️ **Add UPSTASH_REDIS_REST_URL and UPSTASH_REDIS_REST_TOKEN from Upstash console**
   `.env.local`
