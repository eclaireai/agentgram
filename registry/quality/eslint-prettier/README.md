# ESLint + Prettier

> Consistent code style enforced at commit time — no more style debates in PRs

## What it does

Set up ESLint with TypeScript rules and Prettier for formatting. Configured to run as a pre-commit hook via lint-staged, so formatting errors never reach the repo.

## Why this recipe exists

Style debates in code review are waste. ESLint catches real bugs (unused vars, missing awaits, type errors). Prettier eliminates formatting debates entirely. Together they keep code readable without thinking.

## Steps (8 total)

1. 🔍 **`find`** → `package.json`  
   Check existing linting setup and TypeScript config

2. ⚡ **`run_command`** → `npm install -D eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin prettier eslint-config-prettier`  
   Install ESLint, TypeScript plugin, and Prettier

3. 📄 **`create_file`** → `.eslintrc.json`  
   Create ESLint config: TypeScript parser, recommended rules, no-unused-vars, no-floating-promises, prettier last

4. 📄 **`create_file`** → `.prettierrc`  
   Create Prettier config: singleQuote, trailingComma all, printWidth 100, tabWidth 2

5. 📄 **`create_file`** → `.prettierignore`  
   Add dist, coverage, node_modules, *.json to prettierignore

6. ✏️ **`modify_file`** → `package.json`  
   Add lint, lint:fix, and format scripts

7. ⚡ **`run_command`** → `npm run lint`  
   Run linter on codebase and fix auto-fixable issues

8. ⚡ **`run_command`** → `npm run format`  
   Format all files with Prettier
