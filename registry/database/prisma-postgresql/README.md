# Prisma + PostgreSQL

> Type-safe ORM with schema-first migrations, seed data, and connection pooling

## What it does

Set up Prisma ORM with PostgreSQL from zero: schema definition, initial migration, seed script, and connection pooling with PgBouncer-compatible connection strings.

## Why this recipe exists

Raw SQL is brittle. Sequelize is verbose. Prisma gives you type-safe queries generated from your schema, so a typo at the database layer is a compile error — not a production bug.

2. ⚡ **`run_command`** → `npm install prisma @prisma/client`

3. ⚡ **`run_command`** → `npx prisma init --datasource-provider postgresql`

4. ✏️ **`modify_file`** → `prisma/schema.prisma`

5. ✏️ **`modify_file`** → `.env`

   Ensure .env is gitignored, add prisma/migrations to version control

7. ⚡ **`run_command`** → `npx prisma migrate dev --name init`  
   Generate and run initial migration

8. 📄 **`create_file`** → `prisma/seed.ts`  
   Create seed script with 3 sample users and posts using createMany

9. 📄 **`create_file`** → `src/lib/prisma.ts`  
   Create singleton PrismaClient with global instance for dev hot reload

10. ✏️ **`modify_file`** → `package.json`  
   Add prisma.seed script pointing to tsx prisma/seed.ts

11. ⚡ **`run_command`** → `npx prisma db seed`  
   Seed the database and verify data
