# Drizzle ORM + SQLite

> Lightweight SQL with Drizzle — perfect for edge functions, Cloudflare Workers, Bun

## What it does

Set up Drizzle ORM with SQLite (via better-sqlite3 or Bun native). Schema-first, fully type-safe, no migration runner required. Ideal for edge deployments and local-first apps.

## Why this recipe exists

Prisma is too heavy for edge. Drizzle generates SQL that runs anywhere — Cloudflare Workers, Bun, Vercel Edge. The API is SQL-first, so if you know SQL you already know Drizzle.

## Steps (10 total)

1. 🔍 **`find`** → `package.json`  
   Check runtime (Node/Bun/Edge) and existing ORM

2. ⚡ **`run_command`** → `npm install drizzle-orm better-sqlite3`  
   Install Drizzle and SQLite driver

3. ⚡ **`run_command`** → `npm install -D drizzle-kit @types/better-sqlite3`  
   Install Drizzle Kit for migrations and type definitions

4. 📄 **`create_file`** → `src/db/schema.ts`  
   Define users and posts tables with Drizzle schema builder

5. 📄 **`create_file`** → `src/db/index.ts`  
   Create and export drizzle(db) client singleton

6. 📄 **`create_file`** → `drizzle.config.ts`  
   Configure Drizzle Kit with schema path and SQLite driver

7. ⚡ **`run_command`** → `npx drizzle-kit generate:sqlite`  
   Generate initial SQL migration

8. ⚡ **`run_command`** → `npx drizzle-kit push:sqlite`  
   Apply migration to database

9. 📄 **`create_file`** → `src/db/seed.ts`  
   Seed database with sample users and posts

10. ⚡ **`run_command`** → `npx tsx src/db/seed.ts`  
   Run seed and verify rows inserted
