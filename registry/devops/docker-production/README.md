# Production Docker Setup

> Multi-stage builds, health checks, non-root user, and secrets — done right

## Why This Exists

Single-stage Docker builds produce images 3x larger and expose dev dependencies. No health checks means Kubernetes and ECS can't detect crashes. Running as root is a security vulnerability. This recipe fixes all three.

## Steps (7 total)

1. 🔍 **Read start script and Node version requirement**
   `package.json`

2. 📄 **Multi-stage: deps (npm ci --frozen-lockfile), builder (npm run build), runner (node:alpine, non-root user, COPY from builder)**
   `Dockerfile`

3. 📄 **Exclude: node_modules, .env, .git, .next/cache, coverage, *.test.ts, *.spec.ts**
   `.dockerignore`

4. 📄 **Compose with app (healthcheck), postgres:16-alpine (with healthcheck), redis:7-alpine, named volumes**
   `docker-compose.yml`

5. 📄 **Production override: restart: unless-stopped, no bind mounts, resource limits**
   `docker-compose.prod.yml`

6. ▶️ **Build production image and verify it works**
   `docker build -t app:latest --target runner .`
   *Expected: Build succeeds*

7. ▶️ **Verify image starts cleanly**
   `docker run --rm app:latest node -e "process.exit(0)"`
