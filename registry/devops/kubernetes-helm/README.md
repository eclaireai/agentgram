# Kubernetes Deployment with Helm

> Deploy any app to K8s with HPA, rolling updates, and secret management

## Why This Exists

Raw Kubernetes YAML is verbose and error-prone. Helm charts are parameterizable, version-controlled, and shareable. This chart handles the production concerns (resource limits, autoscaling, secrets) that raw YAML tutorials skip.

## Steps (6 total)

1. ▶️ **Scaffold a new Helm chart**
   `helm create charts/app`

2. ✏️ **Set image.repository, resources.requests/limits, autoscaling.enabled, ingress.enabled, replicaCount**
   `charts/app/values.yaml`

3. 📄 **Create HPA scaling on CPU (70%) and memory (80%) with min 2 / max 10 replicas**
   `charts/app/templates/hpa.yaml`

4. 📄 **Create ExternalSecret pulling DATABASE_URL and API keys from AWS Secrets Manager**
   `charts/app/templates/externalsecret.yaml`

5. ✏️ **Add liveness/readiness probes, security context (non-root), and rollingUpdate strategy**
   `charts/app/templates/deployment.yaml`

6. ▶️ **Validate chart syntax**
   `helm lint charts/app`
   *Expected: 1 chart(s) linted, 0 chart(s) failed*
