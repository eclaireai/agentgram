# Terraform AWS Infrastructure

> VPC, ECS Fargate, RDS Postgres, and ALB — production AWS in code

## Why This Exists

Clicking through the AWS console creates infrastructure you can't reproduce or audit. Terraform makes infrastructure reproducible, reviewable (git diff), and destroyable. One command to spin up prod, one to tear it down.

## Steps (8 total)

1. 📄 **Create Terraform root module with AWS provider and required_version constraints**
   `infrastructure/main.tf`

2. 📄 **Create VPC module: 2 AZs, public subnets (ALB), private subnets (ECS, RDS), NAT Gateway**
   `infrastructure/vpc.tf`

3. 📄 **Create ECS Fargate cluster, task definition, service with auto-scaling on CPU/memory**
   `infrastructure/ecs.tf`

4. 📄 **Create RDS PostgreSQL 16, Multi-AZ, automated backups (7 days), encrypted storage**
   `infrastructure/rds.tf`

5. 📄 **Create ALB with HTTPS listener (ACM cert), HTTP→HTTPS redirect, and target group health checks**
   `infrastructure/alb.tf`

6. 📄 **Define variables: environment, region, app_name, db_instance_class, min/max_capacity**
   `infrastructure/variables.tf`

7. 📄 **Output: alb_dns, rds_endpoint, ecr_repository_url, ecs_cluster_name**
   `infrastructure/outputs.tf`

8. ▶️ **Initialize Terraform and preview changes**
   `terraform init && terraform plan`
   *Expected: Plan: N to add, 0 to change, 0 to destroy*
