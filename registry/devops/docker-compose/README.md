# Docker + Docker Compose

> Containerize any Node.js app with multi-stage builds and compose for local dev

## What it does

Add Docker to a Node.js project: multi-stage Dockerfile (builder + production), docker-compose.yml for local development with PostgreSQL and Redis, and .dockerignore.

## Why this recipe exists

Multi-stage builds produce images 3x smaller than single-stage. The dev compose file means any new team member can be up and running in one command — no more "works on my machine."

## Steps (7 total)

1. 🔍 **`find`** → `package.json`  
   Check start script and Node version requirements

2. 📄 **`create_file`** → `Dockerfile`  
   Create multi-stage Dockerfile: deps stage (npm ci), builder stage (npm run build), production stage (node:alpine with dist only)

3. 📄 **`create_file`** → `.dockerignore`  
   Add node_modules, .env, .git, dist, coverage, tests to .dockerignore

4. 📄 **`create_file`** → `docker-compose.yml`  
   Create compose file with app, postgres:15, and redis:7 services, named volumes, healthchecks

5. 📄 **`create_file`** → `docker-compose.prod.yml`  
   Create production override: pull from registry, no bind mounts, restart: always

6. ✏️ **`modify_file`** → `.env.example`  
   Add POSTGRES_USER, POSTGRES_DB, REDIS_URL vars

7. ⚡ **`run_command`** → `docker build -t app:test --target production .`  
   Build production image and verify it succeeds
