# API Rate Limiting

> Per-IP and per-user rate limiting with Redis-backed sliding window

## What it does

Add rate limiting to an Express API: global limit (100 req/min per IP), authenticated user limit (1000 req/min), and endpoint-specific limits. Redis-backed for distributed deployments.

## Why this recipe exists

Without rate limiting, one misbehaving client can take down your API. This recipe implements sliding window rate limiting — stricter than token bucket, fairer than fixed window. Redis backing means it works across multiple API instances.

3. 📄 **`create_file`** → `src/middleware/rateLimiter.ts`

4. ✏️ **`modify_file`** → `src/app.ts`  
   Apply globalLimiter to all routes, authLimiter to authenticated routes

5. ✏️ **`modify_file`** → `src/routes/auth.ts`

7. ⚡ **`run_command`** → `npm test`
