# OpenAPI / Swagger Docs

> Auto-generated, interactive API docs from your TypeScript types

## What it does

Add OpenAPI 3.0 documentation to an Express API using tsoa or swagger-jsdoc. Generates /docs with Swagger UI and a machine-readable openapi.json at /api-docs.

## Why this recipe exists

API docs that live outside the code go stale. tsoa generates docs from your TypeScript decorators — so if it compiles, the docs are current. Every endpoint, every type, every example: always accurate.

## Steps (9 total)

1. 🔍 **`find`** → `package.json`  
   Check Express version and TypeScript config

2. ⚡ **`run_command`** → `npm install tsoa swagger-ui-express`  
   Install tsoa and Swagger UI

3. ⚡ **`run_command`** → `npm install -D @types/swagger-ui-express`  
   Install Swagger UI types

4. 📄 **`create_file`** → `tsoa.json`  
   Create tsoa config: entryFile, noImplicitAdditionalProperties, outputDirectory

5. 📄 **`create_file`** → `src/controllers/UserController.ts`  
   Convert UserController to tsoa @Route, @Get, @Post decorators with @Body and @Path params

6. ✏️ **`modify_file`** → `src/app.ts`  
   Register swagger-ui-express at /docs and mount generated routes

7. ✏️ **`modify_file`** → `package.json`  
   Add tsoa:spec and tsoa:routes scripts, run them in prebuild

8. ⚡ **`run_command`** → `npm run tsoa:spec && npm run tsoa:routes`  
   Generate OpenAPI spec and routes

9. ⚡ **`run_command`** → `curl localhost:3000/api-docs | head -5`  
   Verify OpenAPI spec is served correctly
