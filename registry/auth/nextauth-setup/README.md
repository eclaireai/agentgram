# NextAuth.js Setup

> Drop-in auth for Next.js: Google, GitHub, email/password, session management

## What it does

Add NextAuth.js to a Next.js app with Google OAuth, GitHub OAuth, credentials provider, and Prisma adapter for session persistence. Production-ready in one session.

## Why this recipe exists

Building auth from scratch in Next.js is a 2-day rabbit hole. NextAuth gives you OAuth, sessions, and JWT in one package — but the initial setup with multiple providers is non-trivial. This recipe does it right.

## Steps (9 total)

1. 🔍 **`find`** → `package.json`  
   Check Next.js version and existing auth setup

2. ⚡ **`run_command`** → `npm install next-auth @auth/prisma-adapter`  
   Install NextAuth and Prisma adapter

3. 📄 **`create_file`** → `app/api/auth/[...nextauth]/route.ts`  
   Create NextAuth handler with Google and GitHub providers

4. 📄 **`create_file`** → `lib/auth.ts`  
   Export auth options and getServerSession helper

5. ✏️ **`modify_file`** → `prisma/schema.prisma`  
   Add Account, Session, User, VerificationToken models for NextAuth adapter

6. ⚡ **`run_command`** → `npx prisma migrate dev --name nextauth`  
   Run NextAuth database migration

7. ✏️ **`modify_file`** → `.env.local`  
   Add NEXTAUTH_SECRET, GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, GITHUB_ID, GITHUB_SECRET

8. 📄 **`create_file`** → `components/AuthButton.tsx`  
   Create sign in/sign out button component using useSession

9. ✏️ **`modify_file`** → `middleware.ts`  
   Add route protection middleware using NextAuth matcher
